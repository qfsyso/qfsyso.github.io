使用FG实现图片压缩#

使用 FunctionGraph 实现图片压缩

对象存储服务（OBS）** 与 函数工作流（FunctionGraph） 完成图片压缩处理。

1. 创建 OBS 桶#

注意事项：必须创建 两个不同的桶：

  • 源桶：保存用户上传的原始图片
  • 目标桶：保存压缩处理后的图片

如果只使用一个桶，上传图片会再次触发函数执行，造成无限循环。

1.1 创建源桶#

  1. 登录 对象存储服务控制台，点击 创建桶。
  2. 在「创建桶」界面填写以下信息：
    • 区域：北京四
    • 桶名称：bucket-input-xxx

      如提示名称已存在，可自定义，但需记住此桶为源桶。

    • 数据冗余存储策略：多AZ存储
    • 默认存储类别：标准存储
    • 桶策略：私有
    • 默认加密：关闭
    • 归档数据直读：关闭
    • 其余参数保持默认。
  3. 点击 立即创建 完成。

1.2 创建目标桶#

  1. 按照 1.1 的步骤再创建一个桶。
  2. 区域及存储类别与源桶保持一致。
  3. 桶名称：bucket-output-xxx

    可自定义，需记住此桶为目标桶。

2. 构建程序#

2.1 创建函数#

  1. 登录 函数工作流控制台。
  2. 区域选择 华北-北京四。
  3. 在左侧导航栏选择 函数 > 函数列表，点击 创建函数。
  4. 填写函数信息：
    • 函数名称：fss_examples_image_thumbnail
    • 委托名称：serverless_trust
    • 运行时语言：Python 3.6
  5. 点击 创建函数。

2.2 上传代码#

  1. 进入 fss_examples_image_thumbnail 函数详情页。
  2. 下载示例代码压缩包：
    • fss_examples_image_thumbnail_eg.zip
  3. 在「代码」页签选择 上传自 ZIP 文件，上传下载的 fss_examples_image_thumbnail_eg.zip。
  4. 上传完成后系统会自动部署，若未自动部署，可手动点击 部署。

2.3 配置函数#

进入 设置 > 常规设置 页签：

  • 函数执行入口：index.handler（无需修改）
  • 执行超时时间：40
  • 内存：256
  • 点击 保存。

设置环境变量#

重要提示：环境变量 不能包含空格，否则无法成功连接到 OBS。

在 设置 > 环境变量 中添加：

键值
output_bucket你的目标桶名称，例如：bucket-output-xxx
obs_endpointobs.cn-north-4.myhuaweicloud.com

3. 添加依赖包#

函数示例代码依赖 pillow 包，需要添加依赖：

  1. 在 fss_examples_image_thumbnail 函数详情页，进入 代码 页签。
  2. 滑动至底部，点击 添加依赖包。
  3. 添加公共依赖包：pillow-7.1.2。

4. 添加事件源#

创建 OBS 应用事件源触发器，实现上传图片时自动触发函数。

  1. 在 fss_examples_image_thumbnail 函数详情页，进入 设置 > 触发器。
  2. 点击 创建触发器，选择：
    • 触发器类型：OBS应用事件源
    • 触发器名称：Image
    • 桶：选择之前创建的 bucket-input-xxx 源桶。
  3. 其他配置项保持默认，点击 确定。

5. 测试图片压缩#

  1. 登录对象存储服务控制台，进入 bucket-input-xxx 桶。
  2. 上传 Home 目录下的 background.jpg 图片。
  3. 上传完成后，OBS 会生成事件触发函数自动运行。
  4. 进入 bucket-output-xxx 桶，即可查看压缩后的图片文件大小。
  5. 也可以在函数详情页 日志 中查看运行日志。

结果验证#

  • 输入：上传到 bucket-input-xxx 的原始图片。
  • 输出：函数自动压缩后，图片存储在 bucket-output-xxx。
  • 效果：原图与压缩后的图片体积差异可在目标桶中直接查看。

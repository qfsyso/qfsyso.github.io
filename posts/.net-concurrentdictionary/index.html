线程安全字典（ConcurrentDictionary）#

APNS 推送逻辑 里维护一个 通话列表，列表里存储每个 UserID 和 ReadID 的 状态值，并且和推送逻辑关联。

定义一个 枚举 表示通话状态值，方便维护。

用一个 线程安全字典（ConcurrentDictionary）来存储 (UserID, ReadID) → 状态值。

每次收到消息时，更新状态表，并根据状态值执行推送逻辑。

代码结构更清晰，避免重复判断。

using System;
using System.Collections.Concurrent;
using System.Threading.Tasks;

public enum CallState
{
    Connecting = 0,       // 连接中
    Connected = 1,        // 已连接
    Ended = 2,            // 已结束
    Canceled = 3,         // 已取消
    Declined = 4,         // 已拒绝
    Accepted = 5,         // 已接听
    NoPrivacyPassword = 6,// 没有设置隐私密码
    Busy = 7,             // 忙线
    NoAnswer = 8,         // 无应答
    CallFailed = 9,       // 通话失败
    CallInterrupted = 10  // 通话中断
}

public class CallSessionManager
{
    // 线程安全字典，维护通话状态 (UserID, ReadID) -> 状态
    private static readonly ConcurrentDictionary<(string UserID, string ReadID), CallState> CallStates 
        = new ConcurrentDictionary<(string, string), CallState>();

    /// <summary>
    /// 更新通话状态
    /// </summary>
    public static void UpdateCallState(string userId, string readId, CallState state)
    {
        CallStates[(userId, readId)] = state;
    }

    /// <summary>
    /// 获取当前通话状态
    /// </summary>
    public static CallState GetCallState(string userId, string readId)
    {
        return CallStates.TryGetValue((userId, readId), out var state) ? state : CallState.Connecting;
    }

    /// <summary>
    /// 推送逻辑
    /// </summary>
    public static void HandlePush(dynamic received, int rsint8, int isIos17, string tkdev, string[] tkdevs)
    {
        // 更新状态表
        var state = (CallState)rsint8;
        UpdateCallState(received.UserID, received.ReadID, state);

        // 根据状态做推送
        if (state == CallState.Connecting)
        {
            if (isIos17 == 0)
            {
                // VOIP 推送
                Task.Run(() => ApnsClient.SendPushAsync(
                    tkdev, 
                    received.Un2rs, 
                    received.Con2rs, 
                    received.AutoID.ToString(),
                    received.UserID,
                    received.MsgState.ToString(),
                    received.MsgType.ToString(),
                    received.Content,
                    received.Un2rs,
                    sandbox: true, 
                    voip: true
                ));
            }
            else
            {
                tkdev = tkdevs[0];
                string con2rs = received.MsgType == 3 ? "邀请进行视频通话" : "邀请进行音频通话";

                Task.Run(() => ApnsClient.SendPushAsync(
                    tkdev,
                    received.Un2rs,
                    con2rs,
                    received.AutoID.ToString(),
                    received.UserID,
                    received.MsgState.ToString(),
                    received.MsgType.ToString(),
                    received.Content,
                    received.Un2rs,
                    sandbox: true
                ));
            }
        }
        else
        {
            // 其他状态可以根据需要扩展推送逻辑
            Console.WriteLine($"[{received.UserID}-{received.ReadID}] 状态更新为：{state}");
        }
    }
}

通话状态值集中在 CallState 枚举中，容易维护。

CallSessionManager 管理所有用户通话状态（线程安全）。

推送逻辑跟状态解耦，以后要加不同状态下的推送只需扩展 switch 或 if。

只发一次推送#

短时间内连续 state == CallState.Connecting#

对同一个 received.ReadID（同一个接收人）

记录最后一次推送时间：#

给 (UserID, ReadID) 存一个 LastPushTime，如果距离上次推送小于某个阈值（例如 3 秒 / 5 秒），就跳过推送。

只允许一个 Connecting 状态推送： 如果 CallState.Connecting 已经推送过，就不要重复推。

在 CallSessionManager 里加一个 推送时间表，并在推送前判断。

using System;
using System.Collections.Concurrent;
using System.Threading.Tasks;

public enum CallState
{
    Connecting = 0,
    Connected = 1,
    Ended = 2,
    Canceled = 3,
    Declined = 4,
    Accepted = 5,
    NoPrivacyPassword = 6,
    Busy = 7,
    NoAnswer = 8,
    CallFailed = 9,
    CallInterrupted = 10
}

public class CallSessionManager
{
    private static readonly ConcurrentDictionary<(string UserID, string ReadID), CallState> CallStates 
        = new ConcurrentDictionary<(string, string), CallState>();

    // 记录最后推送时间
    private static readonly ConcurrentDictionary<string, DateTime> LastPushTimes 
        = new ConcurrentDictionary<string, DateTime>();

    private static readonly TimeSpan PushCooldown = TimeSpan.FromSeconds(5); // 限制5秒内只推一次

    public static void UpdateCallState(string userId, string readId, CallState state)
    {
        CallStates[(userId, readId)] = state;
    }

    public static CallState GetCallState(string userId, string readId)
    {
        return CallStates.TryGetValue((userId, readId), out var state) ? state : CallState.Connecting;
    }

    public static void HandlePush(dynamic received, int rsint8, int isIos17, string tkdev, string[] tkdevs)
    {
        var state = (CallState)rsint8;
        UpdateCallState(received.UserID, received.ReadID, state);

        if (state == CallState.Connecting)
        {
            // 判断短时间是否已经推送过
            var key = received.ReadID.ToString();
            var now = DateTime.UtcNow;

            if (LastPushTimes.TryGetValue(key, out var lastTime))
            {
                if (now - lastTime < PushCooldown)
                {
                    Console.WriteLine($"[{key}] 在 {PushCooldown.TotalSeconds} 秒内已推送过，跳过。");
                    return; // 不重复推送
                }
            }

            LastPushTimes[key] = now; // 记录最新推送时间

            if (isIos17 == 0)
            {
                // VOIP 推送
                Task.Run(() => ApnsClient.SendPushAsync(
                    tkdev,
                    received.Un2rs,
                    received.Con2rs,
                    received.AutoID.ToString(),
                    received.UserID,
                    received.MsgState.ToString(),
                    received.MsgType.ToString(),
                    received.Content,
                    received.Un2rs,
                    sandbox: true,
                    voip: true
                ));
            }
            else
            {
                tkdev = tkdevs[0];
                string con2rs = received.MsgType == 3 ? "邀请进行视频通话" : "邀请进行音频通话";

                Task.Run(() => ApnsClient.SendPushAsync(
                    tkdev,
                    received.Un2rs,
                    con2rs,
                    received.AutoID.ToString(),
                    received.UserID,
                    received.MsgState.ToString(),
                    received.MsgType.ToString(),
                    received.Content,
                    received.Un2rs,
                    sandbox: true
                ));
            }
        }
        else
        {
            Console.WriteLine($"[{received.UserID}-{received.ReadID}] 状态更新为：{state}");
        }
    }
}

改动点

新增 LastPushTimes 记录每个 ReadID 的最后推送时间。

在 CallState.Connecting 推送前判断，如果在 5 秒内已经推送过，就跳过。

时间间隔可配（PushCooldown）。

json status 处理#

可以用 C# 的 System.Text.Json 或 Newtonsoft.Json 来修改 status 字段。

用 System.Text.Json (内置库，.NET Core/5+/6+ 推荐)#

using System.Text.Json;

// 原始 JSON
string json = "{\"status\":3,\"callID\":\"198D042E-6460-443A-86C1-93BC1EE799BF\",\"duration\":0}";

// 反序列化为 JsonDocument
using var doc = JsonDocument.Parse(json);
var root = doc.RootElement;

// 创建可修改的对象
var obj = new
{
    status = 0, // 改为 0
    callID = root.GetProperty("callID").GetString(),
    duration = root.GetProperty("duration").GetInt32()
};

// 序列化回字符串
string newJson = JsonSerializer.Serialize(obj);
Console.WriteLine(newJson);

输出：

{"status":0,"callID":"198D042E-6460-443A-86C1-93BC1EE799BF","duration":0}

用 Newtonsoft.Json (更灵活)#

using Newtonsoft.Json.Linq;

// 原始 JSON
string json = "{\"status\":3,\"callID\":\"198D042E-6460-443A-86C1-93BC1EE799BF\",\"duration\":0}";

// 解析为 JObject
var jObj = JObject.Parse(json);

// 修改 status
jObj["status"] = 0;

// 转成字符串
string newJson = jObj.ToString();
Console.WriteLine(newJson);

输出同样是：

{
  "status": 0,
  "callID": "198D042E-6460-443A-86C1-93BC1EE799BF",
  "duration": 0
}

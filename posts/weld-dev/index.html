WLED项目概览#

WLED项目超简单指南#

控制彩色灯条变颜色、闪花样。

一、先准备好“魔法工具”#

要玩WLED，得先凑齐这些零件，就像玩积木要先准备积木块一样：

  • ESP板：核心“指挥官”，有两种——推荐用ESP32（新出的，更好用），别买ESP8266（快淘汰了）。
  • 彩色灯条：要选能和“WS2812B”兼容的（比如WS2812、SK6812这些型号），它能亮红、绿、蓝等各种颜色。
  • 5V电源：给灯条“喂电”的，就像给玩具装电池，灯条没它就不亮。
  • 电线：把零件连起来的“小管子”；如果电线超过半米长，还要准备一个“电平转换器”（不然灯可能会闪来闪去）。
  • 按钮（可选）：想按一下就换灯效的话就准备，不想也没关系，用手机或电脑控制更方便。

二、把零件“手拉手”连起来（接线）#

就像拼乐高要按图纸拼，接线也有规则，别接错哦！

1. 连“指挥官”（ESP板）和灯条#

  • 用ESP32板：找灯条上标着“数据”的线（一般是彩色的），接在ESP32板上写着“GPIO16”“GPIO4”“GPIO13”的地方（选一个就行）。
  • 用ESP8266板：灯条的“数据线”接在板上“GPIO2”的地方（很多板上也会标“D4”，找这个标记也能接）。

2. 一定要接“地线”！#

ESP板和灯条上都有“GND”的接口，用电线把这两个接口连起来！不接的话，灯条可能不亮，还会弄坏零件。

3. 给灯条“喂电”#

把5V电源的正负极，对应接在灯条的“5V”和“GND”接口上。如果灯条很长（比如超过5米），要在灯条中间多接一个电源，不然后面的灯可能会变暗。

4. 可选：装个按钮#

想按按钮换灯效的话，把按钮的一端接ESP板的“GPIO0”（如果是ESP32板，就接“IO17”），另一端接“GND”就行。

三、刷程序#

ESP板空着的时候不会干活，得给它装WLED的“程序”（就像给玩具装“大脑”，让它知道怎么控制灯）：

两种装程序的方法：#

  1. 简单版（直接用现成程序）：
    去WLED官网下载最新的“固件文件”，用电脑连ESP板，跟着官网的图文步骤，把文件“刷”进板里（就像给手机装APP一样简单）。
  2. 复杂版（自己改程序）：
    如果想自己改灯效的代码，就要装“Arduino”软件，编好程序再刷进板里。

装成功的信号：#

程序刷好后，灯条的前30个灯会亮橙色，这就代表“装对啦！”

四、用手机/电脑控制灯条#

“指挥官”有了“大脑”，接下来就能用手机或电脑指挥灯条啦！

1. 先连ESP的“临时WiFi”#

程序刷好后，ESP板会发出一个叫“WLED-AP”的WiFi信号。用手机或电脑连这个WiFi，密码是“wled1234”。

2. 打开控制页面#

连好WiFi后，在手机/电脑的浏览器里输“4.3.2.1”，就能看到控制灯的页面了；也可以试试输“wled.me”，有时候也能打开。

3. 连家里的WiFi（更方便）#

每次连“WLED-AP”太麻烦？可以让ESP板连家里的WiFi：

  1. 点控制页面上的“齿轮图标”（这是“设置”按钮），选“Wifi Setup”。
  2. 输你家WiFi的名字和密码，点“Save & Connect”。
  3. 之后手机/电脑再连家里的WiFi，就能直接控制灯条，不用每次连“WLED-AP”了。

4. 找ESP的“地址”#

连好家里WiFi后，让大人帮你去路由器的“设备列表”里找“WLED”的设备，记下来它的“IP地址”（比如“192.168.1.100”），以后输这个地址就能控制灯了。

五、玩灯！换颜色、变花样#

打开控制页面后，你就能当“灯光魔法师”啦：

  • 选“预设效果”：里面有呼吸灯、流水灯、彩虹色等几十种效果，点一下灯条就会跟着变。
  • 调颜色：页面上有个“颜色盘”，点你喜欢的颜色，灯条就会变成那个颜色。
  • 调亮度：滑动“亮度条”，能让灯变亮或变暗，晚上别调太亮哦，会晃眼睛。

六、升级（更新程序）#

如果WLED出了新功能，要给ESP板更程序，就像给手机更APP一样：

1. 简单版（无线更）#

  1. 打开控制页面，进“设置”，输默认密码“wledota”，把“OTA locked”的勾去掉，保存并重启ESP板。
  2. 再进“安全设置”，选“Manual OTA update”，上传新的固件文件。
  3. 更完后，记得再把“OTA locked”的勾打上，这样更安全。

2. 多设备更（如果有好几个ESP板）#

让大人帮你找“multi-update”脚本（在WLED的“tools”文件夹里），改一下里面的IP地址，就能一次给所有ESP板更程序，不用一个一个来。

小提醒#

  1. 数据线别太长：超过半米要加“电平转换器”，超过10米的话，灯可能会闪，不好控制。
  2. 安全第一：电源和电线要选对，最好加“保险丝”（像电线的“保护盾”，电流太大时会断开，防止烧坏零件）。

doc#

https://github.com/wled/WLED

项目解析网站： https://deepwiki.com/wled/WLED https://ReadmeX.com/wled/WLED 安装： https://install.wled.me/

项目类型：嵌入式/物联网固件（WLED），主要运行在 ESP8266/ESP32 上，为地址灯条（如 WS2812B、SK6812 等）提供丰富的灯效与网络控制。 主要语言与栈： C/C++（Arduino/ESP-IDF 生态） 平台与构建：通常使用 PlatformIO（platformio.ini），也可能有 Arduino IDE 支持 Web 前端资源：node_modules 存在，说明内置 Web UI 的构建（压缩/打包）通过 Node 工具链完成 典型特性：本地/网络控制、效果引擎、传感器/用户模块扩展、MQTT/Home Assistant 集成、文件系统 SPIFFS/LittleFS 托管 Web UI

目录结构要点#

wled00/ src/：固件核心 C/C++ 源码，包含主循环、网络栈适配、效果引擎、设备抽象、配置存取等 data/：Web UI 前端静态资源（图标、像素图等），通过上传到设备文件系统提供 UI usermods/：用户模块扩展（即插即用的功能插件），如传感器（温湿度、光照、气压等）、输入设备、显示屏、风扇、继电器、多路输出、音频反应等 lib/ESP8266PWM/：第三方库（此处为 PWM 控制库），用于特定硬件控制能力 include/：头文件/公用定义（可能是项目级别配置或宏） tools/、pio-scripts/：构建脚本、打包/上传辅助（如自动生成版本、打包资源、文件系统镜像） node_modules/：前端工具链依赖（构建压缩 UI、打包图标、CSS/JS 处理等） build_output/：构建产物（固件 bin、map、release）

开发与构建流程（典型）#

使用 PlatformIO： 常见命令： pio run：编译固件 pio run -t upload：编译并通过串口上传 pio run -t uploadfs 或 -t uploadfs: 上传 data/ 到设备文件系统（LittleFS/SPIFFS，具体看配置） 在 platformio.ini 中定义多个环境（env）：区分 ESP8266/ESP32、不同闪存布局、不同特性开关 前端资源： 通常由 npm 脚本打包压缩 UI 再嵌入或上传至文件系统镜像 node_modules 表明已有依赖，实际构建脚本一般在 package.json（如 npm run build） 产物： build_output/firmware 下可找到 .bin 固件文件，用于 OTA 或串口刷写 release 中可能是已打包好的多平台固件

扩展机制与可定制点#

usermods：每个子目录一个模块，按需启用/编译进固件（常通过配置宏或 platformio.ini 的 build_flags） 传感器与外设：如 AHT10、BH1750、BME280/68X、INA226、Si7021、PIR、LDR、RTC、SD 卡等 功能插件：MQTT、Home Assistant、音频反应、PWM 风扇、七段数码管、OLED/TFT 显示、继电器控制等 UI 资源与图标：wled00/data 下的 icons-ui、pixart、pxmagic 等子目录
